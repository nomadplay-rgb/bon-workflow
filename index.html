Step-by-step

STEP 1 ~ 5 세부 내용

각 스텝에서 실제로 무엇을 하는지, 유형(병원소식/홍보게시물)에 따라 달라지는 부분과 사용자에게 질문이 발생하는 지점을 함께 표시했습니다.

1
STEP 1

확인 질문

추가 인터뷰 없이 딱 3가지만 한 번에 묻고 바로 STEP 2로 진행한다.

  • 유형 — 병원소식 / 홍보게시물 (주제로 추정 가능하면 먼저 제안, 센터신설·장비도입·진료과 홍보는 모두 "홍보게시물")
  • 주제/소재 — 어떤 내용인지 한 줄
  • 이미지 — 첨부 가능 여부 (없으면 Unsplash 대체)
병원소식 → 톤 A(팩트 정보 위주) 자동 적용 홍보게시물 → 톤 B(소비자 공감형 광고 카피) 자동 적용
✔ 사용자 질문 있음 — 이 스텝의 3가지 질문이 파이프라인 전체에서 유일하게 사용자에게 직접 묻는 지점이다. 톤을 직접 언급하면("전문적으로"/"더 공감가게") 그 즉시 반대 톤으로 전환.
2
STEP 2

리서치 · 기획

2-A(리서치 4종 병렬) → 2-B(기획, 페이지 수·구성 확정) 순서로 진행. 사용자에게 묻지 않고 자동 실행.

2-A 리서치 (병렬 4종)
  • 2-A_01 경쟁 동물병원 (항상) — 해마루·VIP·로얄동물메디컬그룹
  • 2-A_02 주요 대학 수의학과 (항상) — SNU·KU·CBNU
  • 2-A_03 의료 용어·논문 자문 (필요 시) — 학술 용어 포함 주제일 때만
  • 2-A_04 인스타그램 인기 키워드 (항상) — 해시태그 조회수·인기도 순 정리
2-B 기획
  • 병원소식 — 4~5페이지 고정 (표지 → 본문 2~3장 → 마무리)
  • 홍보게시물 — 4~8페이지 유동 (표지·마무리 고정, 본문만 조절)
  • 카피 없이 페이지별 개요 한 줄씩만 확정
  • BON 차별화 방향·인기 키워드 배치 위치까지 결정
질문 없음 — 4개 리서치 결과 종합 후 자동으로 구성안 확정, 바로 STEP 3으로 이동.
3
STEP 3

카피

3-A(소비자 실제 언어 리서치, 항상 실행) → 3-B(카피 작성) 순서로 진행. 글자수 초과 시 즉시 축약.

3-A 소비자 실제 언어 리서치
  • 네이버 카페 — "우리 애가 자꾸 ~해요" 식 증상 묘사
  • 인스타 해시태그 — 반응 좋았던 경쟁사·인플루언서 패턴
  • 네이버 지식인·블로그 — 질문형 헤드라인 소재
3-B 카피 작성 (톤 A/B)
  • A(팩트 정보 위주) — 결론→근거, 수치·용어 정확 표기, 감정 수사 최소화
  • B(소비자 공감형) — 3-A 소재를 헤드라인 재료로 사용, 증상→원인, 질문형·대비형 헤드라인
질문 없음 — 단, 진행 중 톤 변경 요청이 들어오면 그 시점에 즉시 반영.
4
STEP 4

HTML 시안 생성

기획안 확정 즉시 전체 슬라이드를 스크롤 가능한 단일 HTML로 만들어 사용자 확인을 받는다.

  • 각 슬라이드 1080×1350px, scale:0.5 미리보기
  • 이미지 없을 시 Unsplash 자동 삽입, 실패 시 플레이스홀더 컬러 사용
  • "시안입니다. 확인 후 수정사항 알려주세요. 완료되면 Figma 파일을 생성합니다." 안내 후 대기
확인 필요 지점 — 질문은 아니지만, 사용자 승인을 받아야 STEP 5(Figma 생성)로 진행 가능.
5
STEP 5

Figma MCP 파일 생성

승인 즉시 실제 슬라이드를 Figma에 생성하고, 이후 수정 요청은 해당 노드만 재수정한다 (5-A~5-D).

  • 5-A 사전 준비 — 신규 파일 생성, Noto Sans KR 폰트 로드
  • 5-B 프레임 좌표 — 1140px 간격 순차 배치, 2회 분할 호출로 토큰 절약
  • 5-C 완료 시 자동 브랜드 검증 — 색상·글자수·세이프존·금지 컬러 점검
  • 5-D 수정 요청 처리 — 전체 재생성 없이 해당 프레임/노드만 수정, B-2 토큰 값만 사용
질문 없음 — 생성 실패 시에만 상황 안내 후 재시도(섹션 F 대응).
