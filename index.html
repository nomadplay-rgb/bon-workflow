STEP 1 확인 질문 — 유형별 전체 4문항

  1. 유형 — 병원소식(이미지 풀블리드형, 좌측 정렬) / 홍보게시물(이미지 박스형, 가운데 정렬). 주제로 추정 가능하면 먼저 제안. ※ 센터 신설·장비 도입·특정 진료과 홍보는 모두 “홍보게시물” 유형.
  2. 카피톤 — 톤 A(팩트 정보 위주) / 톤 B(소비자 공감형 광고 카피). 유형별 기본 제안값 존재, 사용자가 다르게 답하면 그대로 적용.
  3. 주제/소재 — 어떤 내용인지 한 줄.
  4. 이미지 — 첨부 가능 여부 (없으면 Unsplash 플레이스홀더 사용).

STEP 2 리서치 · 기획

2-A 리서치 에이전트 (병렬 실행)

  • 2-A_01 경쟁 동물병원 (항상) — 해마루·VIP·로얄동물메디컬그룹
  • 2-A_02 주요 대학 수의학과 (항상) — 서울대·건국대·충북대
  • 2-A_03 의료 용어·논문 자문 (필요 시) — 학회·논문 근거 확인
  • 2-A_04 인스타 인기 키워드 (항상) — 해시태그 조회수·인기도 순 정리

2-B 기획 에이전트

  • 병원소식: 4~5p 고정 구조로 페이지 수 확정
  • 홍보게시물: 4~8p 사이 유동적으로 결정 (본문만 조절)
  • 페이지별 개요 한 줄씩 정리 (카피 문장은 미작성)
  • BON 차별화 방향·인기 키워드 배치 위치 확정

STEP 3 카피

3-A 소비자 실제 언어 리서치 (항상 실행)

  • 네이버 카페 — "우리 애가 자꾸 ~해요" 증상 묘사
  • 인스타그램 해시태그 — 반응 좋은 카피 패턴
  • 네이버 지식인/블로그 후기 — 질문형 헤드라인 소재

3-B 톤별 작법 비교

톤 A (팩트) — 결론→근거 순서, 수치·용어 정확 표기, 감정 수사 최소화
"장마철 습도 상승, 피부병·외이염 주의보"

톤 B (공감형) — 보호자 실제 표현을 헤드라인 소재로, 증상→원인 순서, 질문형 헤드라인
"자꾸 발을 핥고 귀를 턴다면, 장마철 습도 때문일 수 있어요"

STEP 4 HTML 시안 생성

기획안 확정 즉시 생성. 작성 전 반드시 references/templates/의 실물 PDF(병원소식_템플릿.pdf 5p, 홍보게시물_템플릿.pdf 6p)를 열어 정렬 방향·로고 위치·배경 구성을 대조.

  • 단일 HTML, 슬라이드당 1080×1350px, scale:0.5 미리보기
  • Unsplash API 자동 삽입 — 실패 시 유형별 지정 플레이스홀더 컬러
  • 생성 후 사용자 확인 요청 → 승인 시 STEP 5 진행

STEP 5 Figma MCP 파일 생성

5-A · 5-B 준비 및 좌표

  • whoami → create_new_file, Noto Sans KR 폰트 로드
  • 프레임: P1=x0, 이후 1140px 간격 가로 배치
  • 생성 순서: 배경→오버레이/존분리→로고·캡션·타이틀→본문→세부요소
  • 토큰 절약 위해 2회 분리 호출 (P1~3 / 나머지)

5-D 생성 후 수정 요청 처리

  • 요청 범위 파악 (텍스트/스타일/레이아웃)
  • 해당 프레임만 재작업 — 전체 재생성 금지
  • 색상·폰트는 브랜드 토큰표 기준값만 사용
  • 글자수 제한 재확인 후 반영
