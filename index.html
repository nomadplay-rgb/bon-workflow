본동물의료센터 카드뉴스 제작
파이프라인 워크플로우

리서치·기획·카피라이팅 에이전트가 병렬로 동시에 움직이며, 수정 가능한 Figma 파일 완성까지 전 과정을 자동으로 진행합니다. 자동화 실행 시 지점 선택 질문으로 시작해, 콘텐츠 유형과 카피톤에 대한 질문이 함께 동반됩니다.

Asset

로고·컬러 에셋 고정

지점별 로고 파일(컬러/화이트)과 브랜드 컬러 토큰이 매 게시물에 동일하게 적용되어, 담당자나 제작 회차가 바뀌어도 브랜드 톤이 흐트러지지 않습니다.

Layout

레이아웃 레퍼런스 고정

실제 인쇄 템플릿의 좌표·구조를 그대로 참조해 제작하므로, 매번 다른 배치·정렬이 나오지 않고 시리즈 전체의 통일감이 유지됩니다.

Tone

카피톤 일관성

병원소식(톤 A)·홍보게시물(톤 B)의 기본 어투 규칙이 정해져 있어, 작성자가 달라도 콘텐츠 성격에 맞는 어조가 유지됩니다.

Accuracy

지점 정보 정확도

전화번호·주소·로고가 지점 선택 질문 하나로 전체 슬라이드에 정확히 반영되어, 지점 혼동으로 인한 오탈고를 원천적으로 방지합니다.

Step-by-step

단계별 세부 내용

단계별 특이사항만 간단히 정리했습니다. 특정 단계를 바로 보고 싶으면 위 워크플로우 박스를 클릭하세요.

00
최우선 · 단독 질문

지점 선택

  • 단독 질문 — 유형 질문과 절대 병합하지 않고, 다른 어떤 질문보다 먼저 묻는다.
  • 이 답변 하나로 로고 파일·전화번호·주소가 이후 모든 프레임에 고정 적용된다.
실제 질문
0. 지점: 안양본동물의료센터 / 수원본동물의료센터
01
지점 확정 직후

확인 질문

  • 유형·주제·이미지 3가지를 한 번에 질문 — 추가 인터뷰 없이 바로 STEP 2로 진행.
  • 주제만 보고 유형 추정이 가능하면 먼저 제안한다 (센터 신설·장비 도입 등은 모두 "홍보게시물").
  • 카피톤은 여기서 묻지 않는다 — STEP 3에서 별도 확인.
실제 질문
1. 유형: 병원소식(좌측 정렬) / 홍보게시물(가운데 정렬)
2. 주제/소재: 어떤 내용인지 한 줄
3. 이미지: 첨부 가능 여부 (없으면 Unsplash 사용)
02
4개 병렬 리서치

리서치 · 기획

  • 리서치 에이전트 4종이 동시에(병렬) 실행 — 경쟁병원 · 대학 수의학과 · 용어자문(필요 시) · 인스타 키워드.
  • 기획 단계에서는 페이지 구성만 확정하고 카피는 아직 쓰지 않는다.
  • 사용자에게는 차별화 방향·키워드·구성을 간략 요약만 보여준다.
03
톤 확정 → 작성

카피

  • 카피 작성 직전에 톤을 질문 — 유형별 기본값이 있지만 자유롭게 변경 가능.
  • 소비자 실제 언어 리서치(네이버 카페·해시태그·지식인)가 카피 작성과 병렬로 함께 돈다.
  • 대화 중 "병원소식이지만 톤 B로" 같은 전환 요청도 즉시 반영.
실제 질문
카피톤: 톤 A(팩트 정보 위주) / 톤 B(소비자 공감형 광고 카피)
04
사용자 확인용

HTML 시안 생성

  • 실물 템플릿을 매번 다시 열어 대조 — 정렬·로고 위치·배경 구성 확인 후 제작.
  • 로고는 반드시 실제 파일을 base64 이미지로 삽입 — 텍스트 대체 금지.
  • 미리보기 축소는 zoom만 사용 (transform:scale() 금지).
05
승인 즉시 실행

Figma MCP 파일 생성

  • 사용자 승인 즉시 실행, 프레임을 2회 분리 호출해 토큰 절약.
  • 완료 시 캔버스·정렬·로고·색상 등 14개 항목 자동 브랜드 검증.
  • 수정 요청은 해당 프레임만 재작업 — 전체 재생성하지 않는다.
